# 架构备忘录：BIC Agent 系统性问题聚类与三个架构级动作

**结论先行**：不重写。编排骨架（L2 事件溯源 / L3 子图 / trial 状态机 / L4）经 2026-07-08
全天 32 个 issue 的实测检验，只暴露 2 个真正的结构性 bug（#5、#21），且均为"语义未定清"
而非"结构错误"；TLC 全腿、retry、ELN gating、阶段门、防注入在黄金链路实测中全部 PASS。
问题 60% 集中在**叙述/LLM 自由度这一层**。据此启动三个架构级动作（非修补）：
**① 叙述层契约化（deterministic-first）② 状态语义规范化 ③ 模型 A/B 评估**。
本备忘录同时是实施令：①② 即刻开工，③ 排在统一验证窗口后。

## 二、为什么不重写（Facts）

- 骨架实测健康：T-main 三轮黄金链路，TLC 推荐/静默 retry/结果呈现/链路推进/ELN gating 全 PASS。
- 60% 问题在同一层；重写不消灭"LLM 说话不靠谱"，只是重打已经打赢的仗。
- 已落地的 20+ 修复以根源型为主（结构闸/一等通道/权威投影/确定性收尾），技术债未恶化。

## 三、动作 ①：叙述层契约化（即刻开工）

**原则：每条用户可见叙述 = f(真实状态)。** 从"prompt 约束 LLM 自由改写"切换到
"**确定性骨架（状态驱动）+ LLM 仅润色**"，关键位置（失败、状态宣称、过渡 turn）纯模板零 LLM。

- 契约文本：`BIC-agent-service/.trellis/spec/backend/L3/narrate-contract.md`（随本备忘录落地，
  DRAFT，合并时 Drake 审）。
- 实施顺序：
  1. 在途 be-tail 批（#28 确定性进度投影 / 过渡 turn 确定性文案 / 输出后处理管线）落地 P0 核心；
  2. 随后一个 S3 **按契约清扫全部剩余叙述点**（objective/plan/tlc/cc/fp/re/query 的每个
     narrate 出口逐一归类：纯模板 / 骨架+润色 / 自由——自由类仅允许闲聊问答）；
  3. 契约不变量进测试：每个 narrate 出口有"状态一致性"断言（叙述提及的参数/状态必须
     等于 DB 真实值）。
- 与 PR#66 的关系：locale 穿透与本契约正交（它管语言，本契约管内容真实性）；实现时对齐挂载点。

## 四、动作 ②：状态语义规范化（即刻开工）

**单一语义**：`trial.status` = 权威生命周期（终态判定唯一来源）；`phase` = 子图内部
recollect/rts/conducting/done 阶段机；`progress` = live 执行明细投影（易失、仅供展示补充）；
`jobs.status` = job 级 rollup；`experiments.stage` = 顶层工作流阶段。任何消费方**禁止**用
非权威字段做生命周期判定（#5/#21/#26 的教训）。

- 实施：先全库审计（BE + portal 全部读取点，file:line + 所用语义 + 是否合规），产出
  `ops/state-semantics-audit-2026-07-09.md`；违规点列修复清单，按批次 S3 修；语义表落
  spec（L2 persistence / FE 契约各一节）。

## 五、动作 ③：模型 A/B（统一验证窗口后）

假设：第 1/3 类中大量症状（横跳/幻觉/重复退化/语言漂移/指令不遵循）是 qwen3.7-plus 特性。
- 方案：同一 bench、同一剧本（T-main 黄金链路 + T-challenge 挑战集），仅换 `DEFAULT_MODEL`
  各跑一轮：qwen3.7-plus（基线，已有数据）vs deepseek-v3 系（DashScope 同 key 可用）
  vs 其他可得模型。指标：findings 数 × 严重度 × 类别分布 + 黄金链路通过步数。
- 产出：数据化的模型选型建议，供 Drake 决策。测试员体系已就位，边际成本一晚。

## 七、风险

- narrate 契约化降低"惊喜感"换取可靠性——demo 语境下正确取舍；闲聊/问答仍保留 LLM 自由度。
- 状态语义收敛可能暴露更多隐性依赖（审计先行即为此）。
- 模型切换涉及成本/延迟/合规维度，A/B 只供数据，决策在 Drake。
