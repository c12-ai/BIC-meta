# S2 任务：TLC 物料准备 modal 交互模型设计方案（只调查+设计，产出给用户 review，不实现）

你是 S2 设计会话。用户在台架实测 #46 branch 的物料 modal 后给出一手观察与三条硬性 UX 要求，要求出一版**完整方案**供其 review，之后才决定如何 update portal#46。**产出 = 一个 meta issue 的方案文档；零代码。**

## 用户一手观察（台架实测 #46 行为，全部当事实对待）

1. 点绿色（库存管）→ 变蓝（选中为实验物料）——符合预期。
2. **再点蓝色 → 蓝色消失且刷新不回，库存管被直接删除**；且用户不能点空白使其变绿——严重：写穿库存、无草稿、无取消，误点两下=真删台账。
3. 点"添加纯品/粗品"后可点空白变蓝（放置纯/粗品）——看起来 ok；但点绿色仍只是变蓝，**没有纯品/粗品区分**。

## 用户的三条硬性要求（方案必须逐条满足并说明如何满足）

1. **维护能力一致性**：该界面能否做耗材维护（绿↔空）？可以有也可以没有，但**必须始终一致**（不能一下能一下不能）。考虑到纯/粗品与实验紧密相关，可以考虑允许用户同时点空白点与绿点来指定位置——否则用户要先去维护界面登记两管位置变绿、再回来选中标记纯/粗品，两趟太复杂。
2. **可取消且恢复原状**：取消时恢复进 modal 前的状态（原来绿的还是绿、原来空的还是空）；绝不出现"原来是绿、点两次变空白"的破坏。
3. **类型先行**：若必须指定纯/粗品，则必须先选类型才能点格子；不出现无类型的空白蓝点。
