# Issues and Pull Requests

Use GitHub-native Issues and Pull Requests in the repository where the work actually belongs.

## Issues

Create issues for:

- Requirements that need follow-up implementation.
- Bugs or regressions.
- Technical debt with concrete acceptance criteria.
- Cross-repo coordination that cannot be handled by one PR.

## Pull Requests

PRs should include:

- What changed.
- Why it changed.
- How it was verified.
- Related PRD, issue, or Project item links.
- Cross-repo dependencies, if any.

## Project Tracking

Add real Issues and PRs to the BIC GitHub Project:

https://github.com/orgs/c12-ai/projects/9

Use:

- `Area = Issues` for issues.
- `Area = PRs` for pull requests.
- `Repo Scope` for the owning repository.
