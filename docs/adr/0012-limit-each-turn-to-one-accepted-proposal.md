# Limit each Turn to one accepted Proposal

Foundation v1 permits multiple rejected candidate adjudications but at most one accepted effectful Proposal for each Turn. A corrected intent after rejection is a new candidate; once one Proposal commits, the Proposal capability closes for that Turn and only non-effectful explanation or projection may continue. The accepted Proposal is a business action and may atomically change multiple workflow entities, append the required ordered compatibility events, and create one or more correlated Outbox Commands. The limit is enforced durably by L2 under concurrency and replay rather than by prompts, Agent State, or process-local locks. This prevents an Agent loop from assembling a partially committed multi-action sequence while leaving future batch semantics available as a separately evidenced contract.
