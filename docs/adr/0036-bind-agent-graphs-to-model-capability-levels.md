# Bind Agent graphs to model capability levels

Foundation will expose exactly two centrally governed, provider-neutral Model Capability Levels in v1: `light` and `complex`. Every hosted Agent graph declares exactly one of them, and Foundation alone maps that level to an approved concrete model when the graph is invoked. Domain Packs cannot add levels. Graphs do not declare arbitrary capability bags or model roles, and graph nodes, Domain code, Skills, prompts, and clients cannot select a provider or model dynamically. Foundation records the graph, declared level, and concrete provider/model for each actual call as Turn-correlated Model Invocation Provenance. The model mapped behind a stable level may evolve without rebinding workflows, while exact provider/model lifecycle pinning remains a separate regulatory or reproducibility Demand Gate.
