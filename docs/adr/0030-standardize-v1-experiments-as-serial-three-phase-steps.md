# Standardize v1 experiments as serial three-phase Steps

V1 standardizes one Experiment Workflow Template: establish an Experiment Objective, create an Experiment Plan containing ordered domain-defined Steps, execute those Steps serially, complete the Plan, and produce a Summary Document. Every Step uses the shared cycle of Preparation/Parameter Design, Experiment Execution, and Result Analysis. A parameterized Step exposes an execution-bearing design and requires the existing Confirmation Gate before execution. A manual Step with no system-generated execution parameters returns a typed `not_required` preparation result, adds no synthetic parameter form or confirmation surface, and continues through the existing human-completion and evidence interaction. Domain Packs own Step kinds and scientific semantics, while the Experiment Workflow Kit owns the reusable lifecycle. In Chemistry v1, Summary Document names the existing deterministic, AI-free ELN report: the Kit owns gating and orchestration, the Pack supplies typed deterministic report data, and Agent finale narration remains separate. Parallel Steps, DAG routing, nested Plans, and a general workflow DSL are deliberately deferred; serial assumptions stay localized behind versioned contracts so a later approved topology can evolve without changing Agent Foundation.
