# Persist input admission before Turn execution

L2 separates durable Input Admission from asynchronous Turn execution. One admission transaction validates and deduplicates the normalized input, applies any deterministic user/system fact transition and CAS, appends its input receipt/Session Event, and inserts the identified Turn as queued. HTTP or MQ acknowledgement occurs only after this transaction commits; an input that fails validation or CAS creates no Turn. A worker later claims the durable Turn, reloads current facts, invokes Foundation, and durably closes the Turn. Queue wait is distinct from the execution deadline, and deterministic admission changes do not consume the Turn's single L3 Proposal acceptance slot. This closes the current crash gap between committed input handling and an in-memory session queue without introducing a durable LangGraph thread.
