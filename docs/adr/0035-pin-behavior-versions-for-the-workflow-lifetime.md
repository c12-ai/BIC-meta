# Pin behavior versions for the workflow lifetime

An Experiment Workflow spans stateless Turns and deployments, so L2 will persist an immutable Workflow Behavior Binding to one exact domain, Domain Pack Manifest version, Experiment Workflow Template version, and registered Behavior Target identity/version before its first domain Agent invocation. The Behavior Target immutably identifies the internal runtime implementation, including a reviewed legacy compatibility target or the new Foundation/Kit/Pack target during coexistence. Every Turn resolves both Pack faces, the Template, and the exact target from that binding; old and new targets may coexist, missing exact versions fail closed, and v1 permits no mutable deployment mapping, per-Turn or per-Step routing, branch fallback, or implicit mid-workflow upgrade. This avoids making deployment routing a hidden workflow authority and keeps scientific behavior, audit, rollout, and rollback coherent.
