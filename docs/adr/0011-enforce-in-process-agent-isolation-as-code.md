# Enforce in-process Agent isolation as code

Foundation v1 remains in the Agent Service process but is isolated by executable contracts rather than convention. L3 and Domain Agent Definitions receive only approved Pure, Read-Only Query, and Proposal ports; they cannot import or receive workflow persistence, concrete L2 services, outbox repositories, or state-changing external clients, while L2-facing Domain Proposal Policies receive only neutral immutable intent/current-fact views and pure decision contracts. Ruff/static lint, import-linter dependency contracts, type and architecture tests, Composition Root tests, and startup manifest validation all run as required CI gates. No database transaction spans model execution, and Turn cancellation does not propagate across an already committed Proposal/Outbox boundary. A separate process or sandbox is deferred until an approved untrusted Skill, MCP, or plugin use case justifies the RPC, streaming, cancellation, and operational cost.
