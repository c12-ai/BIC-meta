# Migrate L3 through vertical slices

The L3 redesign uses seven independently verifiable delivery groups: compatibility and architecture gates; Durable Turn, Proposal, and Outbox primitives; rolling-summary produce/load closure; the complete internal Chemistry happy path; remaining Chemistry behavior behind disabled routing; Biology portability plus contract freeze and full Chemistry rerun; and one-time cutover plus legacy deletion. DL-01 decomposes each group into one-to-three-engineer-day implementation children with a pre-committed binary acceptance check and rollback boundary.

Code merges incrementally to `main` behind disabled internal routing rather than a long-lived migration branch. A named validation run cannot fall back by Turn, Step, Plan shape, model output, unsupported behavior, or failure, and the migration permits no effectful shadow run, dual write, duplicate Proposal acceptance, or second command path. Chemistry precedes Biology because Biology exploration has no real examples; real Chemistry migration provides a better provisional contract than imagined Biology behavior. Biology still challenges that contract before freeze, and every Chemistry slice reruns afterward. Once all gates pass, every unfinished old-path workflow on the bench and at field sites completes or resets, the default switches once, and legacy routing and code are deleted. Expand, migrate, contract applies only if `a1` field data or `orin-tail` data must survive; resettable environments may use reset-and-seed.
