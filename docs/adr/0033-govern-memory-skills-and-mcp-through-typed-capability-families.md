# Govern Memory, Skills, and MCP through typed capability families

Future Memory, Skill, and MCP support shares one demand-gated admission and least-authority grant policy but not a generic runtime extension interface. The Composition Root binds each approved provider to the narrowest typed seam: Foundation governs execution, context, budgets, provenance, and classification; the Experiment Workflow Kit exposes phase-scoped consumption points; Domain Packs and trusted static Skills declare requirements and contribute domain semantics without receiving raw stores, transports, credentials, or registries. Memory reads enter as lower-authority context, model-selected Memory writes require a typed Memory Proposal, and every MCP operation is admitted and classified individually as Query, Proposal intent, or an External Command executable only by the outbox executor. This decision defines extension boundaries but does not add a production Memory Store, general Skill runtime, MCP gateway, dynamic loader, or untrusted extension execution to Foundation v1.

The v1 rolling-conversation-summary slice is separate from General Agent Memory. It completes the existing stored projection's producer and loader so context survives the 50-event reconstruction window, replay uses the stored projection, consumption can be disabled for fallback, and source Session Events remain unchanged. It introduces no generic Memory provider, Store, or model-selected write contract.
